# 论文功能图生成提示词

## 统一风格要求

| 项目 | 要求 |
|---|---|
| 图像类型 | 论文/毕业设计用 UML 流程图、活动图、时序图 |
| 配色 | 黑白灰，不使用彩色 |
| 背景 | 纯白背景 |
| 线条 | 黑色细线，清晰、规整 |
| 字体 | 中文清晰可读，类似宋体/黑体，不能模糊 |
| 风格 | 学术论文插图风格，简洁、正式、无装饰 |
| 比例 | 横向图片，适合插入 Word 论文 |
| 输出 | 高清 PNG，文字必须清晰 |

## 01 登录注册活动图

生成一张黑白 UML 活动图，标题为“登录注册活动图”。

流程从开始节点进入，依次为：

1. 用户进入系统
2. 选择注册或登录
3. 填写账号与密码信息
4. 判断“输入信息是否合法”
   - 若否，流向“提示重新填写”，然后返回填写账号与密码信息
   - 若是，继续下一步
5. 判断“用户名或密码是否有效”
   - 若否，流向“提示重新登录或更换用户名”，然后返回填写账号与密码信息
   - 若是，继续下一步
6. 生成用户 Token
7. 进入系统首页
8. 结束节点

要求使用标准活动图符号：开始黑点、圆角矩形表示操作、菱形表示判断、箭头连接流程、结束节点为靶心圆。整体居中排版，黑白配色，文字清晰。

## 02 登录注册时序图

生成一张黑白 UML 时序图，标题为“登录注册时序图”。

参与对象从左到右依次为：

1. 用户
2. 前端
3. FastAPI
4. MongoDB
5. JWT

消息流程如下：

1. 用户 -> 前端：填写注册信息
2. 前端 -> FastAPI：POST /auth/register
3. FastAPI -> MongoDB：检查用户名是否存在
4. MongoDB -> FastAPI：返回查询结果
5. FastAPI -> MongoDB：写入用户信息与密码哈希
6. FastAPI -> 前端：注册成功
7. 用户 -> 前端：输入账号密码登录
8. 前端 -> FastAPI：POST /auth/login
9. FastAPI -> MongoDB：读取用户与密码哈希
10. FastAPI -> JWT：生成访问 Token
11. FastAPI -> 前端：返回 Token 与用户名
12. 前端 -> 用户：进入系统首页

要求每个对象顶部用矩形框表示，下方是垂直生命线。调用消息用实线箭头，返回消息用虚线箭头。黑白风格，文字清晰。
